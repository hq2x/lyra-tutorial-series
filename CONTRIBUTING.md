# 贡献指南

感谢你对 UE5 Lyra 教程的关注！我们欢迎各种形式的贡献。

## 如何贡献

### 报告问题

如果你发现了错误或有改进建议：

1. 检查 [Issues](https://github.com/lobsterchen/ue5-lyra-tutorial-docs/issues) 确保问题未被报告
2. 创建新 Issue，清晰描述问题
3. 如果可能，提供复现步骤或截图

### 修正错误

1. Fork 本仓库
2. 创建分支：`git checkout -b fix/your-fix`
3. 修改文档
4. 提交：`git commit -m 'Fix: 描述你的修正'`
5. 推送：`git push origin fix/your-fix`
6. 创建 Pull Request

### 改进内容

1. Fork 本仓库
2. 创建分支：`git checkout -b improve/your-improvement`
3. 改进文档（增加示例、优化表述等）
4. 提交：`git commit -m 'Improve: 描述你的改进'`
5. 推送：`git push origin improve/your-improvement`
6. 创建 Pull Request

### 添加新内容

如果你想贡献新的章节或补充内容：

1. 先创建 Issue 讨论你的想法
2. 等待维护者反馈
3. 按照上述流程提交 PR

## 写作规范

### Markdown 格式

- 使用标准 Markdown 语法
- 代码块指定语言：````cpp`、````bash` 等
- 使用相对链接引用项目内文件

### 文档结构

每篇文章应包含：

1. **标题和元信息**
2. **目录**（章节较长时）
3. **引言**（本文目标、前置知识）
4. **正文**（清晰的章节划分）
5. **代码示例**（完整可运行）
6. **总结**
7. **参考资料**
8. **下一步**（推荐阅读）

### 代码规范

- C++ 代码遵循 UE 编码规范
- 添加必要的注释
- 示例代码应可编译运行
- 避免过于复杂的示例

### 图片和媒体

- 图片放在 `assets/images/` 目录
- 使用描述性文件名：`01-modular-actor-lifecycle.png`
- 优化图片大小（< 500KB）
- 提供图片说明

## Pull Request 流程

1. **提交前检查**：
   - 拼写和语法
   - Markdown 格式正确
   - 链接有效
   - 代码可运行

2. **PR 描述**：
   - 清晰说明改动内容
   - 关联相关 Issue（如有）
   - 说明测试情况

3. **等待审核**：
   - 维护者会尽快审核
   - 可能需要修改
   - 保持耐心和友好

## 行为准则

- 尊重他人
- 保持专业
- 建设性沟通
- 开放心态
